Why we like Ember

  • Fast and snappy user experience
  • Well organized structure
  • Easy to understand and learn
  • Works nice with servers thanks to Ember Data

But is it really true?

What was wrong?

  • Expensive rerendering - prior to v1.13.0
  • What are views for at all?
  • Lots of magic - not as much as on My Little Pony but still
  • Compability of Ember, Ember Data, Ember CLI versions

The Ember Project

Starting from version 2.0 coordinated relase of Ember with releases of the main ecosystem tools maintained by the core team. This will allow painless update of related tools in the project.

  • Ember CLI
  • Ember Data
  • Liquid Fire
  • List View

1.13.0 is the new 2.0...




almost

The new and shiny rendering engine. 



                { enabled: true, name: "Tom Dale",
                  details: "A True Scotsman" }
            
                <div class="item">
                  {{#if enabled}}
                    <p>It's Enabled</p>

                  <p>{{name}}</p>

Changes and deprecations

  • No more views.
  • Ember.select gone.
  • Checkbox view gone.
  • Item controller with {{each}}.

Changes and deprecations

New syntax for bind-attr

                <div {{bind-attr class="isActiveUser"}}></div>

                // Need to be changed to:

                <div class={{if isActiveUser 'is-active-user'}}></div>

Changes and deprecations

One-way values by default

                {{!-- title is a mutable two-way binding --}}
                {{my-component title=model.name}}

                {{!-- title is just an (immutable) value --}}
                <my-component title={{model.name}} />

New stuff => helpers and components

  • New components life cycle hooks like: didInitAttrs, didUpdateAttrs and more
  • No more required dashes for helpers
  • Ability to access services in helpers

New stuff => component attrs property

  • Way to distinguish component own properties from passed values
  • Accessible through attrs property
    •                     <my-component title={{model.name}} />
                    // available with 
                    this.attrs.title
                    // instead of 
                    this.get('title')

New stuff => each-in helper

                let items = {
                  "Item 1": 1234,
                  "Item 2": 3456
                };

                {{#each-in items as |key value|}}
                  <p>{{key}}: {{value}}</p>
                {{/each-in}}

New stuff => get helper

                let items = {
                  "Item 1": 1234,
                  "Item 2": 3456
                };

                {{get items 'Item 1'}}
                // or
                {{get items somePathReturningKey}}

New stuff => mut helper

Way to modify parent value with one way binding 
                // before 1.13.0 
                {{my-counter count=activatedCount}}

                // but we still want to update parent value

                <my-counter count={{mut activatedCount}} />

One more thing...

Drop support for IE8!!!

How to upgrade

  • Start with moving to v1.13.0.
  • Fix tons of deprecations you will get.
  • Update all packages/plugins
  • Change all views into components
  • Update to 2.0

How to upgrade - tools

  • mixonic/ember-cli-deprecation-workflow
  • Plugins with deprecated features like bind-attr or view

What about Ember Data?

Changes to find

  • Currently overloaded.
  • store.find(type)
  • store.find(type,id)
  • store.find(type, {query})
  • find is deprecated and replaced with better named methods

What about Ember Data?

New methods for quering server/storage

Async from server/store Sync from store Query server
Single Record findRecord(type,id) peekRecord(type, id) queryRecord(type, {query})
All Records findAll(type) peekAll(type) query(type, {query})*

Me

Why we like Ember

  • Fast and snappy user experience
  • Well organized structure
  • Easy to understand and learn
  • Works nice with servers thanks to Ember Data

But is it really true?

What was wrong?

  • Expensive rerendering - prior to v1.13.0
  • What are views for at all?
  • Lots of magic - not as much as on My Little Pony but still
  • Compability of Ember, Ember Data, Ember ClI versions

The Ember Project

Starting from version 2.0 Coordinated relase of Ember with releases of the main ecosystem tools maintained by the core team. This will allow painless update of related tools in the project.

  • Ember CLI
  • Ember Data
  • Liquid Fire
  • List View

1.13.0 is the new 2.0...

The new and shiny rendering engine.

                  details: "A True Scotsman" }
            
                <div class="item">

Changes and deprecations

  • No more views.
  • Ember.select gone.
  • Checkbox view gone.
  • Item controller with {{each}}.

Changes and deprecations

New syntax for bind-attr

                <div {{bind-attr class="isActiveUser"}}></div>

                // Need to be changed to:

                <div class={{if isActiveUser 'is-active-user'}}></div>

Changes and deprecations

One-way values by default

                {{!-- title is a mutable two-way binding --}}
                {{my-component title=model.name}}

                {{!-- title is just an (immutable) value --}}
                <my-component title={{model.name}} />

New stuff => helpers and components

  • New components life cycle hooks like: didInitAttrs, didUpdateAttrs and more
  • No more required dashes for helpers
  • Ability to access services in helpers

New stuff => each-in helper

                let items = {
                  "Item 1": 1234,
                  "Item 2": 3456
                };

                {{#each-in items as |key value|}}
                  <p>{{key}}: {{value}}</p>
                {{/each-in}}

New stuff => get helper

                let items = {
                  "Item 1": 1234,
                  "Item 2": 3456
                };

                {{get items 'Item 1'}}
                // or
                {{get items somePathReturningKey}}
